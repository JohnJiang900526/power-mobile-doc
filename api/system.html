# 系统方法 API

系统的方法很多，下面只展示几种常用的方法。

# 1. 调用二维码

// 1.1在vue的程序中调用
new window.PowerApp.system.ScanQRCode((result) => {
  // result 是二维码扫描的结果
  this.address = result;
});


// 1.2在vue的程序中iframe嵌入页面的调用
var that = this;
window.parent.PowerApp && 
new window.parent.PowerApp.system.ScanQRCode(function (result) {
  // result 是二维码扫描的结果
  that.address = result;
});

# 16. DialogPage 打开页面

不带流程

// 1. 在vue的程序中调用
window.PowerApp.tools.OpenDialogPage({
  title: "合同验收",
  type: "innerweb", // innerweb, outerweb
  url: "/static/flow/test.html",
  params: {
    name: "123",
    age: 20
  },
  queryParams: {
    key1: "01",
    key2: "02"
  },
  close: function(res) {
    console.log(res);
  }
});

// 2 在vue的程序中iframe嵌入页面的调用
window.parent.PowerApp && 
window.parent.PowerApp.tools.OpenDialogPage({
  title: "合同验收",
  type: "innerweb", // innerweb, outerweb
  url: "/static/flow/test.html",
  params: {
    name: "123",
    age: 20
  },
  queryParams: {
    key1: "01",
    key2: "02"
  },
  close: function(res) {
    console.log(res);
  }
});

// title 显示的标题
// type 类型 站外页面(outerweb 和url要一直匹配) 站内页面(innerweb 和url要一直匹配)
// url 页面地址 站外地址 站内地址
// params 传进页面的参数 打开的页面内 window下面必须有setData方法
// queryParams url连接传递参数 queryParams设置以后，
// 打开页面的链接上会拼上参数 比如 '/static/flow/test.html?key1=01&key2=02'
// close 关闭页面的回调

# 20. __$$debug 判断开发环境

一般打包之后的代码为生产环境，vue打包之前的代码为开发环境。

生产环境"false", 开发环境"true"

项目上的开发人员一般很少用此API，但是封装开发人员是经常用的。

// 1. 在vue的程序中调用
let debug = window.__$$debug;
console.log(debug); // 生产环境"false", 开发环境"true"


// 2 在vue的程序中iframe嵌入页面的调用
var debug = window.parent.__$$debug;
console.log(debug); // 生产环境"false", 开发环境"true"

# 32. WeixinLocation

微信环境、企业微信环境中 地理位置获取

注意

由于企业微信不提供精确的地理位置信息，

所以在开发PHO页面的时候，需要根据企业

微信提供的经纬度信息，反向解析详细地理

位置，反向解析可以依靠百度地图或者高德

地图,具体的方法，参考百度地图或者高德

地图的API，在此不再赘述。

// 1. 在vue中调用
const Location = window.PowerApp.tools.WeixinLocation();
this.Location = Location;
